How Graphs Are Generated

The AI processes raw crime data to create visualizations:

  1. Data Collection: Aggregates crime counts from NYPD reports, as shown in the Data Sources table.
  2. Normalization: Adjusts counts based on precinct population (e.g., crimes per 100,000 residents).
  3. Visualization: Uses Canvas API to plot bar graphs (crime counts) or line graphs (trends over time).

Example: For Robbery in Precinct 12 (15 incidents, 100,000 residents) and Precinct 10 (10 incidents, 80,000 residents), the AI normalizes to 15 and 12.5 per 100,000, then plots bars or lines using the Canvas API, as implemented in the JavaScript code.

AI Deployment Recommendations

The AI uses crime data and historical patterns to recommend resource allocations (e.g., patrol units, detective teams) for specific areas like the Financial District or Residential Zone. See the Deployment Page for details.

Process:

  • Data from NYPD reports and external factors (e.g., weather) are analyzed, as shown in the Data Sources table.
  • Confidence levels (e.g., 89% for Financial District) are calculated using historical trends and recent data.
  • Recommendations are generated, such as deploying 2 patrol units from 18:00-02:00 based on a 73% increase in robbery patterns.

How AI Marks Hotspots on the Map

The AI identifies and marks crime hotspots on the map using the following process:

  1. Data Source: The AI uses crime data from the Data Sources table, specifically NYPD Crime Reports [1] for crimes like Murder, Felony Assault, Rape, and Robbery.
  2. Filtering: The AI filters the data based on user selections in the dashboard filters (e.g., Borough: Bronx, Crime Type: Property Crimes, Time Period: Last 7 Days). For example, to mark Murder hotspots, the AI queries the dataset for incidents where "Crime Type = Murder" and "Date" is within the selected time frame.
  3. Geospatial Analysis: The AI aggregates crime counts by precinct and maps them to geographic coordinates. For instance, a hotspot at 30% top, 40% left corresponds to a high concentration of crimes in Precinct 12, calculated using a clustering algorithm (e.g., DBSCAN) with a crime density threshold of 10 incidents per square kilometer.
  4. Visualization: Hotspots are marked with red star icons on the map. The position is calculated by mapping precinct centroids to the map's pixel coordinates, scaled to the map's dimensions (e.g., 30% top, 40% left for a precinct centroid).

Example: For Murder in Precinct 12 on 2025-05-24, the AI identified 2 incidents (see Data Sources table). After filtering for "Murder" and normalizing by population (2 per 100,000 residents), the AI marked a hotspot at the precinct's centroid if the crime density exceeded the threshold.
